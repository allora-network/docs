# Basic Usage

## What You'll Learn
- How to interact with the Allora Network at a foundational level
- Essential operations for querying on-chain inference data
- Basic staking operations including delegating to reputers
- Understanding network participant roles and their interactions

## Overview

**The Allora Network is a sophisticated ecosystem designed to facilitate various participants, including inference workers, forecasters, reputers, and validators, each playing a crucial role in the network's functionality and integrity.** Spinning up these different network participants involves a deep understanding of the network's architecture and protocols.

### Network Complexity vs Accessibility

**Despite the complexities involved in the setup and operation of different participants, interacting with the Allora Network on a basic level is straightforward.** Here are some ways to get started:

### Key Network Participants

Understanding the ecosystem participants:
- **Inference Workers**: Submit AI model predictions to the network
- **Forecasters**: Predict the performance of other participants
- **Reputers**: Evaluate and score the accuracy of submissions
- **Validators**: Maintain network security and consensus

### Why Start with Basic Operations?

Basic interactions help you:
- **Learn gradually**: Build understanding without complex setup requirements
- **Test functionality**: Verify network connectivity and operations
- **Explore data**: Understand network activity and participant behavior
- **Prepare for advanced roles**: Foundation for becoming a network participant

## Core Network Interactions

### Querying an Inference On-chain

**Purpose**: Interacting with the Allora Network also involves querying data of existing topics on-chain. This can be efficiently done using the Allorad CLI tool. The CLI tool provides a command-line interface to interact with the network, enabling users to retrieve on-chain data seamlessly.

**Learning Path**: Follow the tutorial [here](/devs/get-started/query-network-data#get-latest-available-network-inferences) to learn how to query an inference on-chain using the `allorad` CLI tool.

**What You'll Accomplish**:
- Retrieve live inference data from the network
- Understand topic-based data organization
- Practice using the allorad CLI for read operations
- Explore network activity and participant contributions

### Delegating Stake to a Reputer

**Purpose**: Users can delegate their stake to a reputer, contributing to the network's overall health and performance. This involves a basic understanding of staking mechanisms and can be done through the `allorad` CLI tool.

**Learning Path**: Follow the tutorial [here](/devs/reference/allorad#delegate-stake-to-a-reputer-for-a-topic) to learn how to delegate your stake to a reputer.

**What You'll Accomplish**:
- Participate in network economics through staking
- Support reputer operations and network quality
- Earn potential rewards from successful delegations
- Practice transaction submission using allorad CLI

## Getting Started Workflow

### Recommended Learning Path

**Phase 1: Setup and Exploration**
1. [Install allorad CLI](/devs/get-started/cli) for network interactions
2. [Create your wallet](/devs/get-started/setup-wallet) for transaction capabilities
3. Query inference data to understand network activity
4. Explore existing topics and their data patterns

**Phase 2: Basic Participation**
1. Fund your wallet using testnet faucet
2. Practice querying different types of network data
3. Delegate small amounts to reputers to understand staking
4. Monitor your delegations and rewards

**Phase 3: Advanced Preparation**
1. Study specific participant roles in depth
2. Consider which network role aligns with your goals
3. Begin setup for your chosen participant type
4. Connect with the community for guidance and support
