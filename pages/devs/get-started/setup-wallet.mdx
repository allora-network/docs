# Setup Wallet

## What You'll Learn
- How to create and manage Allora Network wallets using the allorad CLI
- Wallet recovery procedures using mnemonic phrases
- Accessing testnet faucet funds for development and testing
- Understanding network configurations including RPC URLs and Chain IDs

## Overview

**Setting up a wallet is the essential first step for interacting with the Allora Network.** This guide covers wallet creation, recovery, funding, and network configuration.

### Why You Need a Wallet

An Allora wallet enables:
- **Transaction execution**: Sign and submit transactions to the network
- **Asset management**: Hold and transfer ALLO tokens
- **Network participation**: Engage as a worker, reputer, or validator
- **Multi-network support**: Compatible across all Allora network deployments

## Prerequisites

- [`allorad` CLI installed](/devs/get-started/cli) - Required for wallet management
- Basic understanding of blockchain wallet concepts
- Secure storage method for mnemonic phrases
- Internet connectivity for network interactions

## Wallet Creation

### Create New Wallet

Follow the instructions [here](/devs/get-started/cli) to install our CLI tool `allorad`, which is needed to create a wallet.

Prior to executing transactions, a wallet must be created by running:

```shell
allorad keys add testkey
```

**Important Notes**:
- Replace `testkey` with your preferred wallet name
- Make sure you save your mnemonic and account information safely
- Keep your mnemonic phrase secure and never share it

**What Happens During Creation**:
- Generates a new cryptographic key pair
- Creates a wallet address compatible with all Allora networks
- Displays mnemonic phrase for recovery purposes

Learn more about setting up keys [here](https://docs.cosmos.network/main/user/run-node/keyring).

<Callout type="info">
Creating a wallet using `allorad` will generate a wallet address for all currently deployed versions of the Allora Chain (e.g. testnet, local, mainnet). 
</Callout>

## Wallet Recovery

### Recover Existing Wallet

To recover a given wallet's keys, run the following command:

```bash
allorad keys add <wallet name> --recover
```

**Recovery Process**:
- Enter your existing mnemonic phrase when prompted
- Confirm the recovery by verifying the generated address
- Wallet will be restored with full functionality

**When to Use Recovery**:
- Moving to a new device or installation
- Restoring wallet after system failure
- Accessing wallet from backup mnemonic

## Network Funding

### Add Faucet Funds

Each network has a different URL to access and request funds from. Please see the faucet URLs for the different networks below:

**Testnet Faucet**: https://faucet.testnet.allora.network/

**How to Request Funds**:
1. Visit the faucet URL above
2. Enter the Allora Wallet address for the account that needs funding
3. Submit the request and wait for confirmation

**Prerequisites**: If you don't have a wallet created yet, follow the instructions above to create one.

### Network Explorer

**Testnet Explorer**: https://explorer.testnet.allora.network/

#### Verify Funding

<Callout>
Check to see that your wallet has been funded after requesting funds from the faucet by clicking the search bar on the top right corner of the explorer UI and entering your account address.
</Callout>

**Verification Steps**:
1. Navigate to the testnet explorer
2. Use the search bar in the top right corner
3. Enter your wallet address
4. Confirm your balance appears correctly

## Network Configuration

### RPC URL and Chain ID

Each network uses a different RPC URL and Chain ID which are needed to specify which network to run commands on when using specific commands on `allorad`. See a list of all RPC URLs and their respective Chain IDs supported today:

#### Testnet Configuration

**RPC URLs**:
- https://rpc.ankr.com/allora_testnet
- https://allora-rpc.testnet.allora.network/

**Chain ID**: `allora-testnet-1`

**Usage Tips**:
- Use these values when running allorad commands that require network specification
- The official Allora RPC endpoint typically provides better reliability
- Keep these configurations handy for development work

### Configuration Best Practices

**Security Considerations**:
- Store mnemonic phrases in secure, offline locations
- Use hardware wallets for mainnet operations when available
- Never share private keys or mnemonic phrases
- Regular backup of wallet information

**Development Workflow**:
- Create separate wallets for different environments (testnet, mainnet)
- Use descriptive names for easy wallet identification
- Keep track of wallet addresses for each network
