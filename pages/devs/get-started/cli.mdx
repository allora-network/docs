# Allora CLI Spec

## What You'll Learn
- How to install and configure the allorad CLI tool for Allora Network interaction
- Essential commands for reading and writing data to the Allora blockchain
- Verification steps to ensure proper installation and functionality
- Overview of available CLI commands and their purposes

## Overview

**Allora provides a CLI tool that allows network participants to perform different functions on the Allora Network:**

### What is allorad?

**`allorad`**: Used to read and write data to the chain, e.g. to create a wallet, create new topics or add/delegate stake to a reputer

### Why Use the CLI?

The allorad CLI enables:
- **Wallet management**: Create and manage Allora Network wallets
- **Topic operations**: Create new topics and manage existing ones
- **Stake management**: Add and delegate stake to reputers
- **Data queries**: Read network state and transaction information
- **Network interaction**: Direct blockchain communication and operations

**Complete Reference**: Refer to the [Allorad Reference](/devs/reference/allorad) section for a full list of `allorad` commands with their explanations

## Prerequisites

### System Requirements

You will need to install `go` to download and use `allorad` successfully.

**Go Installation**: To install Go, follow one of the recommended methods below or consult the [official Go documentation](https://go.dev/doc/install) for the correct download for your operating system. The command-line instructions are based on standard installation locations, but you may customize them as needed.

**Additional Requirements**:
- Terminal or command prompt access
- Internet connectivity for downloading the CLI
- Basic familiarity with command-line operations

## Installing `allorad`

### Installation Command

```Text bash
curl -sSL https://raw.githubusercontent.com/allora-network/allora-chain/main/install.sh | bash -s -- v0.8.0

```

**Installation Process**:
- Downloads the latest allorad binary from the official repository
- Installs it to your system's PATH for global access
- Sets up necessary permissions and configurations

### Successful Installation Output

A **successful** installation should output the following line:

```bash
YYYY-MM-DD hh:mm:ss (N MB/s) - '/tmp/allorad' saved [116706514/116706514]
```

**What This Means**:
- Shows download completion with timestamp
- Indicates successful binary download (file size in bytes)
- Confirms installation file was saved to temporary location

## Verifying Installation

### Version Check

After installation, verify that `allorad` is correctly installed and ready to interact with the Allora Network by running:

```
allorad version
```

**Expected Behavior**:
- Returns the installed version number
- Confirms CLI is properly installed and accessible
- Validates the installation was successful

### Command Overview

`allorad` supports general Cosmos SDK and Tendermint commands. You can run the tool to see a list of commands with explanations of what they do:

```text
$ allorad

  query       Querying subcommands
```

### Key Command Categories

**Essential Commands for New Users**:
- **`keys`**: Manage your application's keys (wallet creation and management)
- **`query`**: Querying subcommands (read network data)
- **`tx`**: Transaction subcommands (write data to the network)
- **`status`**: Query remote node for status (network connectivity check)

**Advanced Commands**:
- **`comet`**: CometBFT subcommands (consensus layer operations)
- **`init`**: Initialize validator and configuration files
- **`start`**: Run the full node (for validator operations)

## Common Use Cases

### Getting Started Workflow

1. **Install CLI**: Follow installation steps above
2. **Verify installation**: Run `allorad version`
3. **Create wallet**: Use `allorad keys` commands
4. **Check network status**: Use `allorad status`
5. **Begin operations**: Query data or submit transactions

### Troubleshooting Installation

**If version command fails**:
- Verify Go is properly installed
- Check if allorad is in your system PATH
- Re-run the installation script
- Consult the [official documentation](https://go.dev/doc/install) for Go setup

**If permissions issues occur**:
- Ensure you have proper system permissions
- Consider running installation with appropriate privileges
- Check firewall settings for download access
